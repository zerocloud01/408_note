# 查找
## 概念
**查找：** 在数据集合中寻找满足某种条件的数据元素的过程称为查找。
**查找表：** 用于查找的数据集合称为查找表，由同一类型的数据元素（或记录）组成。
**关键字：** 数据元素中唯一标识该元素的某个数据项的值，使用基于关键字的查找，查找结果应该是**唯一的**。

### 对查找表的常见操作
- 查找符合条件的数据元素
- 插入/删除某个数据元素

静态查找表只需要进行操作1，而动态需要都进行。

### 查找算法的评价指标
查找长度：需要对比关键字的次数
平均查找长度$ASL$：所有查找过程中进行关键字比较次数的平均值。
$ASL = \sum_i^n P_i C_i$
$P_i$ 为查找到元素 $i$ 的概率，$C_i$ 表示查找 $i$ 元素的查找长度

## 二叉排序树BST

## 平衡二叉树 AVL
树上任一节点的左右子树高度差不超过 $1$

**结点的平衡因子** $=$ 左子树高 $-$ 右子树高

## 红黑树RBT
**优点：** 插入/删除时候不会破坏“红黑”特性，无需频繁调整形态。调整也只需要常数级时间。

性质：
- 每个节点是红色或者黑色
- 根节点为黑色
- 叶节点黑色（空节点）
- 红节点不相邻
- 任意节点到任一叶节点的简单路径，黑节点数目相同

如此可以保证最长路径最多为最短路径的两倍。
> 全黑/红黑交替

### 插入操作
插入节点默认是红色。
插入后若是违反性质，则有三种情况与应对方案：
#### 判断节点是根节点
根节点变黑

#### 判断节点的叔叔是红色

## 散列表
即Hash表
散列表：一种数据结构。特点是：可以根据数据元素的关键字计算出它在散列表中的存储地址。
散列函数：$Addr = H(key)$，构建了 关键字 $\to$ 存储地址 的映射关系。

### 冲突
在散列表中插入一个数据元素时，需要根据关键字的值确定其存储地址，若该地址已经存储了其他元素，则称此情况为“冲突”

**同义词：** 发生冲突的关键字为同义词。

构造更适合的散列函数，以减少冲突。

### 散列函数的构造
设计散列函数时需要注意：
- 定义域必须涵盖**所有可能出现的**关键字。
- 值域不能超出地址范围。
- 为尽可能减少冲突，散列函数计算出的地址应该尽可能均匀分布。 
- 尽可能简单，能快速计算出任一关键字对应地址。
#### 除留余数法
$H(key) = key \mod p, p$ 为质数
散列表长度为 $m$，则取 $p \le m$

**适用场景：** 关键字为整数时通用。

#### 直接定址法
$H(key) = key$ 或 $a \times key + b$
$a,b$ 为常数，此方法计算简单，且不会冲突。但是在关键字分布不均匀时空位较多。

**适用场景：** 关键字分布基本连续

#### 数字分析法
选取数码分布较为均匀的若干位为addr

如手机号，一些位数分布的概率不一定相同，而一些位机会均等，则选取均等的这若干位为addr。

**适用场景：** 关键字集合已知且部分数码位分布均等。

#### 平方取中法
取关键字平方值的中间几位作为addr

**适用场景：** 关键字每位取值都不够均匀

### 解决冲突的方法
冲突无法避免，以下是解决冲突的办法：

#### 拉链法
每个地址是一个链表
将同义词放在同一个链表中

#### 开放定址法
设置偏移序列 $d$，每次冲突则查看偏移后的地址 $H_1 = (H(key) + d_i$\% m，
偏移有以下设置方法
- **线性探测法** $d_i = 0,1,2,\ldots, m-1$
- 平方探测法 $d_i = 0^2,1^2,-i^2,2^2,-2^2,\ldots,k^2,-k^2, k\le m/2$
- 双散列法 $d_i = i \times hash_2(key)$
- 伪随机法 $d_i$ 设置为一个伪随机序列

每次查找有重复，则下一偏移元素，直到找到空单元失败。
**对于删除操作**，删除的元素不能简单的置为空，而应该打上标记，防止查找失败。
